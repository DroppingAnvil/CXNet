Here are the core requirements that might cause problems debugging
NetworkContainer - The base of every transmission, resigned by each hop, contents are not trustable, SIGNED BY LAST TRANSMITTER
NetworkEvent - Within NetworkContainer for transmission over any network, Holds event information, CXPath and more, ALWAYS SIGNED BY ORIGIN this allows cryptographic verification to permissions or access control
NetworkEvent.d - ACTUAL PAYLOAD, this data will be either signed by ORIGIN or E2E encrypted by ORIGIN, errors on decrypt are expected for peers without permission

When any event/transmission of any kind is received, if it is a CX style packet, it will immediately be put in an InputBundle so that the event blob and tracking data, ex socket data can be tracked along internal event flow

CXHELLO is the only event that gets to bypass full CX cryptospecs and it is because it is used for pinging in local networks to peers that might not even exist on CXNET
When CX Objects are written to disk ex Node, Blockchain events, they are stored in their transmitted PGP blob form so that they can be reused/retransmitted with proper origin sig
